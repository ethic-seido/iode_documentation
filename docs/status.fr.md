# Statut du **service**

La page de statut permet de vérifier rapidement si les services publics d'ES/IODE fonctionnent normalement, si un incident est en cours ou si une maintenance a été publiée. Elle est utile lorsqu'une recherche ne répond pas comme prévu, lorsqu'un assistant IA semble indisponible ou avant de relancer une requête importante.

```text
Statut: https://esiode.statuspage.io/
```

## Composants suivis

La page de statut publique indique l'état des composants exposés, notamment **Web** et **AI**. Le composant Web concerne l'accès au site et aux parcours visibles dans le navigateur. Le composant AI concerne les fonctions qui dépendent de traitements d'intelligence artificielle, par exemple certains assistants, analyses ou générations.

Une dégradation du composant AI peut donc affecter une fonction avancée sans empêcher la consultation de pages publiques. À l'inverse, un incident Web peut rendre l'accès général instable même si les services de calcul restent disponibles.

## Lire un incident

Lorsqu'un incident est publié, vérifiez la date, le composant concerné, le niveau d'impact et les messages de suivi. Pour un usage scientifique, il est préférable d'éviter de tirer des conclusions sur l'absence de résultats pendant une période d'incident: une indisponibilité peut limiter la recherche, l'actualisation des contenus ou les fonctions assistées.

## Recommandations pour les équipes

Si ES/IODE est utilisé dans un protocole de veille, de revue bibliographique ou de préparation rédactionnelle, notez la date et l'heure des recherches importantes. En cas d'incident, relancez les requêtes critiques après le retour à la normale et conservez la mention de l'interruption dans vos notes de travail lorsque cela peut influencer la reproductibilité.
