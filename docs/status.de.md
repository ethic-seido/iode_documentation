# **Servicestatus**

Die Statusseite zeigt schnell, ob die öffentlichen Dienste von ES/IODE normal funktionieren, ob ein Vorfall läuft oder ob eine Wartung veröffentlicht wurde. Sie ist hilfreich, wenn eine Suche nicht wie erwartet reagiert, ein KI-Assistent nicht verfügbar scheint oder eine wichtige Abfrage erneut gestartet werden soll.

```text
Status: https://esiode.statuspage.io/
```

## Überwachte Komponenten

Die öffentliche Statusseite meldet den Zustand der sichtbaren Komponenten, darunter **Web** und **AI**. Die Web-Komponente betrifft den Zugriff auf die Website und die im Browser sichtbaren Arbeitsabläufe. Die AI-Komponente betrifft Funktionen, die von KI-Verarbeitung abhängen, etwa Assistenten, Analysen oder Generierungsaufgaben.

Eine Störung der AI-Komponente kann also erweiterte Funktionen beeinträchtigen, ohne die Anzeige öffentlicher Seiten zu verhindern. Umgekehrt kann ein Web-Vorfall den allgemeinen Zugriff instabil machen, auch wenn Rechendienste verfügbar bleiben.

## Einen Vorfall lesen

Wenn ein Vorfall veröffentlicht wird, sollten Datum, betroffene Komponente, Auswirkungsgrad und Folgemeldungen geprüft werden. Für wissenschaftliche Arbeit sollten fehlende Ergebnisse während eines Vorfalls nicht vorschnell interpretiert werden: Eine Störung kann Suche, Inhaltsaktualisierung oder assistierte Funktionen beeinflussen.

## Empfehlungen für Teams

Wenn ES/IODE für Monitoring, Literaturübersichten oder Schreibvorbereitung genutzt wird, sollten Datum und Uhrzeit wichtiger Suchen notiert werden. Bei einem Vorfall sollten kritische Abfragen nach der Wiederherstellung erneut ausgeführt und die Unterbrechung in Arbeitsnotizen erwähnt werden, wenn sie die Reproduzierbarkeit beeinflussen kann.
