# Estado del **servicio**

La página de estado permite comprobar rápidamente si los servicios públicos de ES/IODE funcionan con normalidad, si hay un incidente en curso o si se ha publicado un mantenimiento. Es útil cuando una búsqueda no responde como se esperaba, cuando un asistente de IA parece no estar disponible o antes de repetir una consulta importante.

```text
Estado: https://esiode.statuspage.io/
```

## Componentes supervisados

La página pública de estado informa del estado de los componentes expuestos, incluidos **Web** y **AI**. El componente Web se refiere al acceso al sitio y a los recorridos visibles en el navegador. El componente AI se refiere a funciones que dependen de procesos de inteligencia artificial, como asistentes, análisis o tareas de generación.

Una degradación del componente AI puede afectar a una función avanzada sin impedir la consulta de páginas públicas. A la inversa, un incidente Web puede hacer inestable el acceso general aunque los servicios de cálculo sigan disponibles.

## Leer un incidente

Cuando se publica un incidente, conviene revisar la fecha, el componente afectado, el nivel de impacto y los mensajes de seguimiento. Para un trabajo científico, es preferible no sacar conclusiones de la ausencia de resultados durante una ventana de incidente: una indisponibilidad puede afectar a la búsqueda, la actualización de contenidos o las funciones asistidas.

## Recomendaciones para equipos

Si ES/IODE se utiliza para vigilancia científica, revisión bibliográfica o preparación de escritura, registre la fecha y la hora de las búsquedas importantes. Durante un incidente, repita las consultas críticas después de la recuperación y mencione la interrupción en las notas de trabajo cuando pueda influir en la reproducibilidad.
